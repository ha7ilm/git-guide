új repó készítése

Hozz létre egy új könyvárat, nyisd meg, majd a
git init
segítségével hozz létre egy új repót.

repó checkout-olása

Az alábbi paranccsal másodpéldányt hozhatsz létre egy helyi repódból:
git clone /a/repó/útvonala
Távoli szerver használata esetén az alábbi lesz a parancs:
git clone username@host:/a/repó/útvonala

munkamenet

A helyi repód három "fából" áll. Az első a Working Directory (munkakönyvtár), ahol a valódi fájljaid vannak. A második az Index, amely az új verzió eltárolása (commit) előtti "felvonulási területként" (staging area) szolgál, és végül a HEAD, amely az utolsó eltárolt verziódra mutat.

hozzáadás & eltárolás

A változtatásokat az alábbival adhatod az Index-hez:
git add <fájlnév>
git add *
Ez az alap git munkafolyamat első lépése. Az összes hozzáadott változtatás új verzióba való eltárolásához használd ezt:
git commit -m "Commit üzenet"
Most a fájl el lett tárolva a HEAD-nél, de még csak helyben, nem a távoli repóban.

a változtatások elküldése

A változtatásaid a repó helyi másolatának HEAD-jénél vannak. Hogy elküldd őket ("push"-old) a távoli repóba, futtasd az alábbit:
git push origin master
A fenti parancsban master-t aszerint írd át, hogy melyik ágat akarod push-olni.

Ha nem egy létező repót klónoztál le, és hozzá akarod csatlakoztatni a repód egy távoli szerverhez:
git remote add origin <szerver>
Most már el tudod küldeni a változtatásaid a kiválasztott távoli gépre.

elágaztatás

Az új ág (angolul "branch") létrehozása arra való, hogy különböző új funkciókat egymástól függetlenül tudjatok fejleszteni. A master ág az alapértelmezett ág, amikor létrehozol egy repót. Használj más ágakat a fejlesztésre, majd vond össze őket (vagyis "merge"-eld) a master ággal, amikor befejezted.

Hozz létre egy új ágat "feature_x" névvel, majd válts át rá az alábbival:
git checkout -b feature_x
Válts vissza a master-re:
git checkout master
Az új ágat így törölheted:
git branch -d feature_x
Az ág nem lesz elérhető mások számára, amíg nem push-olod a távoli repóba:
git push origin <branch>

frissítés & összevonás

Ha a helyi repód az utolsó commit-ra akarod frissíteni, az alábbival...
git pull
...lehívhatod (fetch) és a jelenlegivel összevonhatod (merge) a távoli változtatásokat.
Hogy egy másik ágat vonj össze az aktív ágaddal (pl. a master-rel), írd be ezt a parancsot:
git merge <branch>
Mindkét esetben a git meg fogja próbálni automatikusan összevonni (auto-merge) a változtatásokat. Sajnos ez nem mindig lehetséges: van amikor összeférhetetlenséget okoz (merge conflict). A te felelősséged, hogy ezt kézzel kijavítsd, belenyúlva azokba a fájlokba, amiknél a git jelzi a problémát. Ezután újra hozzá kell őket adni az Index-hez:
git add <fájlnév>
Az összevonás előtt az alábbi módon kaphatsz egy előnézetet a változtatásokról:
git diff <forrás branch> <cél branch>

címkézés

Javasolt, hogy hozz létre címkéket ("tag"-ek) a kiadott szoftververziókhoz. Ez egy ismert koncepció, amit az SVN-ben is létezik. Hogy létrehozz egy 1.0.0 nevű címkét, hajtsd végre ezt:
git tag 1.0.0 1b2e1d63ff
A 1b2e1d63ff karaktersorozat egy commit azonosítójának első tíz karaktere. (Azt szeretnéd, hogy erre a commit-ra vonatkozzon a címke.) A commit azonosítót megszerezheted, ha belenézel abba, ami most következik...

napló

A repó történetét legegyszerűbb formában így nézheted meg:
git log
Hozzáadhatsz egy csomó paramétert, hogy a log olyan legyen, amilyennek szeretnéd. Ha csak egy adott szerző commit-jait szeretnéd látni:
git log --author=bob
Ha mindent nagyon tömören, egy sorban akarsz:
git log --pretty=oneline
Ha ASCII art-ot szeretnél, amely kirajzolja a fát az összes ággal, megjelölve a címkékkel és az ágak neveivel:
git log --graph --oneline --decorate --all
Ha csak azt akarod látni, hogy mely fájlok változtak:
git log --name-status
Ez csak pár lehetséges paraméter. Ha a többit is látni szeretnéd:
git log --help

a helyi változások visszacsinálása

Ha esetleg valamit elrontottál, ami persze soha nem történhet meg ;), vissza tudod állítani a változtatásaid az alábbi parancssal:
git checkout -- <fájlnév>
Ez a munkakönyvtárba visszateszi a HEAD-ben lévő tartalmat. Meg lesznek tartva azonban azok a változtatások, amelyek már hozzá lettek adva az Index-hez, illetve az előző commit óta keletkezett új fájlok is.

Ha inkább minden helyi változtatást és commit-ot kidobnál, letöltenéd a legújabb commit-okat a szerverről és azt sueretnéd, hogy a master ág a legfrissebb commit-ra mutasson, akkor ezt kell tenned:
git fetch origin
git reset --hard origin/master

hasznos tippek

Beépített git GUI:
gitk
Színes git kimenet:
git config color.ui true
A logban mindig egy sor legyen egy commit:
git config format.pretty oneline
Fájlok interaktív hozzáadása az Index-hez:
git add -i
